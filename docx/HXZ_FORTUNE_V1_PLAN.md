# HXZ Fortune v1 版本规划（面向普通用户）

最后更新：2026-03-29（v1.1 需求已冻结）
适用阶段：第一大版本（v1.x）
目标用户：普通用户自助使用（结论直白、少术语、偏行动建议）

---

## 一、产品目标（v1 总目标）

在同一服务器体系下，提供一个可直接使用的命理推演产品，让用户通过输入出生信息快速获得：

1. 八字基础盘与五行强弱概览
2. 某时间段内的行动建议（适合做什么、规避什么）
3. 易懂的结果表达（降低术语负担）

v1 的定位是“可用 + 好懂 + 可执行”，不追求专业深度展示。

---

## 二、版本边界与输入精度策略

### 输入精度分级（贯穿 v1）

1. 大概模式：出生日期（可无具体时辰）
2. 标准模式：出生年月日时 + 性别 + 阴历/阳历选择
3. 进阶模式：标准模式 + 出生地（用于后续更精确换算）

### v1 内策略

1. v1.1 起支持：阴历/阳历选择 + 大概/标准两档
2. v1.3 起支持：进阶模式（加入出生地）
3. 所有版本都必须展示“精度说明”，避免用户误解结果确定性

---

## 三、版本规划（v1.1 ~ v1.5）

## v1.1 - 可用首发版（MVP）

发布目标：完成从输入到结果的最短闭环

### 功能任务

- 登录策略：免登录直接使用，预留后续登录入口（用于未来跨设备同步）
- 输入表单：姓名（可选）、性别、出生日期、出生时辰（可选）、阴历/阳历选择
- 精度模式：大概模式 / 标准模式
- 缺失时辰策略：
  - 默认先产出“不含时辰”的基础结论
  - 同页提示“补全时辰可升级精度”，并说明可能变化项
- 结果页：
  - 八字信息（简版）
  - 五行占比（木火土金水）
  - 整段建议：宜做、慎做、一句总结
- 时间段推演：
  - 默认区间为未来 30 天
  - 允许用户任意选择起止日期
  - 区间长度上限为 180 天
  - 拆分规则为连续 7 天分段，最后不足 7 天单独成段
  - 每段输出“宜做、慎做、一句提醒 + 4 维度简表（工作学习/财务/人际/健康）”
  - 4 维度简表采用三档表达：低 / 中 / 高
  - 若分段结论与整段不一致，明确提示“本段偏离整体趋势”
- 历史记录：本地保存最近 5 次分析（仅本机）

### UI 任务

- 结构：单页流程（输入 -> 加载 -> 结果）
- 视觉风格：温和陪伴型语气与表达，避免命令式文案
- 视觉：延续 iOS 风格，强调清晰层级与可读性
- 结果卡片：
  - 顶部固定免责声明（结果仅供参考）
  - 顶部结论卡（一句话结论）
  - 五行图示卡（可视化占比）
  - 行动建议卡（宜/慎）
  - 分段建议卡（按 7 天块展示）
  - 精度升级提示卡（在基础结论后展示）
- 交互反馈：加载中、失败重试、结果生成成功提示
- 移动端：优先适配 375 宽度，保证无横向滚动

### 定义完成（DoD）

- 用户在 1 分钟内可完成一次分析
- 每次分析都能生成可读结论和建议
- 缺失时辰时，系统仍可返回基础结论并给出升级提示
- 自定义日期区间可正常拆分为 7 天块并稳定输出
- 出现分段与整段冲突时，会显示“本段偏离整体趋势”
- 术语出现时有简短解释
- 移动端关键流程可用

---

## v1.2 - 体验优化版（表达更易懂）

发布目标：让结果更“像建议”，不是“像报告”

### 功能任务

- 文案分层：先结论后依据（默认收起专业依据）
- 建议模板化：
  - 工作学习
  - 财务决策
  - 人际沟通
  - 作息健康
- 时间段推演增强：支持 7 天 / 30 天 / 自定义区间
- 风险提示：增加“仅供参考，不替代专业决策”

### UI 任务

- 新增“阅读模式切换”：简明模式 / 详细模式
- 结果页加目录锚点：结论、五行、时间段建议
- 优化按钮文案与状态（生成中、已完成、重试）
- 空状态与错误状态统一风格

### 定义完成（DoD）

- 普通用户首屏可读懂结论
- 用户能区分“建议”和“依据”
- 时间段建议可按不同区间稳定生成

---

## v1.3 - 精度升级版（加入出生地）

发布目标：引入更高精度计算条件并做差异提示

### 功能任务

- 输入新增出生地（城市级）
- 精度模式新增“进阶模式”
- 结果页新增“精度标识”与“与标准模式差异提示”
- 计算失败兜底：无法精确换算时自动退回标准模式并提示

### UI 任务

- 输入区改为分组表单：基础信息 / 精度选项 / 时间段
- 出生地输入支持联想或常用城市快捷选择
- 在结果页顶部固定显示本次使用的精度模式
- 对差异内容使用高亮提示（仅展示关键差异）

### 定义完成（DoD）

- 用户可完成进阶模式输入并成功出结果
- 进阶与标准的差异有可读解释
- 失败退回机制可用且无中断

---

## v1.4 - 稳定与复盘版（连续使用能力）

发布目标：提高复用性，支持用户回看与对比

### 功能任务

- 历史记录升级：最近 20 次，支持重命名与删除
- 结果对比：允许选择两次结果进行对比（重点看行动建议变化）
- 时间段推演支持“本月/下月”快捷入口
- 增强异常处理：输入校验、计算超时、结果为空兜底

### UI 任务

- 新增“历史记录抽屉”
- 新增“对比视图”（左右或上下布局，移动端单栏）
- 表单校验提示前置（提交前即提示）
- 提升移动端触控区（按钮最小点击区 >= 40px）

### 定义完成（DoD）

- 用户可稳定复用同一工具进行多次分析
- 历史与对比功能在手机端可用
- 异常场景有明确反馈，不出现空白页

---

## v1.5 - v1 收口版（质量与一致性）

发布目标：统一体验并准备进入 v2 专业化扩展

### 功能任务

- 全链路文案统一（输入、结果、错误、免责声明）
- 结果可信度提示（基于输入完整度给出低/中/高）
- 新手引导：首次使用 3 步提示
- 运营基础：埋点统计（生成成功率、重试率、常用时间段）

### UI 任务

- 统一组件样式规范：按钮、标签、提示、卡片
- 首次进入显示引导浮层（可关闭）
- 结果页增加“下一步建议入口”（如重新分析、切换模式）
- 统一加载动画与反馈样式

### 定义完成（DoD）

- 主要页面视觉与交互一致
- 新用户首次完成率达到可接受水平
- 具备进入 v2 的数据基础

---

## 四、v2 预留方向（本阶段不做）

1. 专业术语展开与完整推导过程
2. 多流派规则配置（可切换算法/解释风格）
3. 专业版报告导出（PDF/长期追踪）
4. 咨询师工作台（批量管理、注释、案例库）

---

## 五、风险与控制

1. 风险：用户将建议视为确定性结论
- 控制：每个结果页固定显示“参考性质”与精度等级

2. 风险：输入不完整导致结果偏差
- 控制：明确提示不同精度模式影响，关键缺失项前置提醒

3. 风险：术语过多影响普通用户理解
- 控制：默认简明模式，专业说明折叠展示

---

## 六、验收主指标（v1 总体验收）

1. 任务完成率：用户能独立完成输入并拿到结果
2. 可理解性：用户能复述“近期宜/慎”的核心建议
3. 可复用性：用户愿意进行第二次分析（历史复用）
4. 稳定性：关键流程错误可恢复，无阻断式失败
