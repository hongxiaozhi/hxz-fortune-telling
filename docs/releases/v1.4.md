# Release Plan: v1.4

最后更新：2026-03-31  
状态：进行中

## 版本目标

增强复盘与复用能力，在不改变现有分析接口顶层结构的前提下，让用户更方便地回看历史结果、做轻量对比，并更快选择常用时间区间。

## 当前版本承接关系

- `v1.3` 已完成输入精度增强、标准 / 进阶模式差异提示和本地 Docker 验证
- `v1.4` 重点承接历史记录增强、结果对比和快捷时间段入口
- 当前继续沿用根目录统一 `docker compose` 运行方式

## 范围边界

本版本包含：

- 增强历史记录摘要信息
- 基于历史记录的轻量结果对比
- 输入页快捷时间段入口
- 历史记录的基础管理能力
- 对比结果的摘要化提示
- 保持现有 `POST /api/fortune/analyze` 顶层结构稳定

本版本不包含：

- 多设备同步
- 账户体系
- 专业咨询工作台
- 跨账号或云端历史记录同步

延期到后续版本：

- 新手引导与全链路文案统一 -> `v1.5`
