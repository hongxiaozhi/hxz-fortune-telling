# Changelog

## v1.3.0 - 2026-03-31

### Added

- 新增出生地输入字段
- 新增标准模式 / 进阶模式
- 新增 `analysis_context` 和 `mode_hint`，用于说明本次分析依据
- 新增 `docs/releases/v1.3.md`

### Changed

- 健康检查版本号升级为 `v1.3.0`
- 结果页新增“本次分析依据”摘要，展示模式、时辰和出生地输入情况
- 输入页新增进阶模式说明文案
- 项目文档中的当前阶段由 `v1.2` 切换到 `v1.3`

### Notes

- 本次正式进入下一个明确版本 `v1.3`
- 当前仍沿用根目录 `docker compose` 与 Nginx 反向代理的运行方式

## v1.2.1 - 2026-03-31

### Changed

- 在页面头部新增当前版本号标识 `v1.2.1`，便于上线后快速确认网页是否已更新

### Notes

- 本次为 `v1.2` 范围内的小补丁，不改变接口结构和结果页主要交互

## v1.2.0 - 2026-03-31

### Added

- 结果页新增“简明模式 / 详细模式”阅读切换
- 新增 `docs/releases/v1.2.md`，用于承接 `v1.2` 的结果表达优化迭代

### Changed

- 结果页默认优先展示一句话结论、整体建议和前几段关键建议
- 详细模式下保留 `v1.1` 的八字简表、五行分布和维度信息
- 结果页新增“阅读引导 / 风险提示 / 使用建议”文案层，降低将结果误读为确定结论的风险
- 项目文档中的当前阶段由 `v1.1` 收尾切换到 `v1.2` 进行中

### Notes

- 本次进入的是下一个明确版本 `v1.2`，没有扩展到 `v1.3`
- 当前后端返回结构保持稳定，第一批迭代聚焦前端阅读体验优化

## v1.1.2 - 2026-03-31

### Changed

- 补强 `POST /api/fortune/analyze` 的 `v1.1.1` 中文结果回归测试，覆盖中文免责声明、整体建议、分段建议与精度提示
- 整理后端测试文件结构，统一夹具、有效载荷构造与断言组织方式
- 同步 README、PROJECT_CONTEXT、ROADMAP 与 `docs/releases/v1.1.md`，使版本状态与线上部署现状一致

### Notes

- 当前 `v1.1` 发布范围已经闭环完成，下一步版本规划以 `v1.2` 为主
- 线上部署继续沿用根目录 `docker compose` 与 Nginx 反向代理，不回退到旧的子项目独立部署方式

## v1.1.1 - 2026-03-31

- 八字简表示例值改为中文干支
- 整体建议和分段建议改为中文表达
- 健康检查版本号升级为 `v1.1.1`

## v1.1.0 - 2026-03-30

### Added

- 健康检查接口
- 运势分析接口基础校验
- 7 天分段建议结果结构

### Changed

- 结果输出统一为可直接消费的 JSON 结构
- 缺失出生时辰时增加精度降级与升级提示
- 前端结果页改为“先结论后细节”的信息顺序
- 输入页、加载态、结果页、历史抽屉统一为稳定的 `v1.1` 主链路
- 补充 Windows 本地 Docker 构建与根目录统一编排验证记录

### Fixed

- 修复前端页面文案乱码和损坏模板导致的主流程不可用问题
- 修复后端文件损坏导致服务无法启动的问题
- 修复线上 Docker 与 Nginx 路由不一致的问题

### Notes

- 当前部署方式已统一为根目录 Docker 编排 + Nginx 反向代理
- 已确认 Windows 本地可通过 `docker compose up -d --build hxz-fortune` 启动并访问修复后的页面版本
