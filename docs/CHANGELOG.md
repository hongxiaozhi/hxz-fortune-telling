### Changed

- 健康检查版本号升级为 `v1.4.1`
- `v1.4` 在历史对比与管理基础上，继续补齐输入回填复用能力

### Notes

- 本次继续保持分析接口顶层结构稳定，重点增强历史记录的复用效率
- 已完成本地 `docker compose up -d --build hxz-fortune` 构建与运行验证

## v1.5.0 - 2026-03-31

### Added

- 新增页面级别新手引导区块，帮助首次用户快速完成必要输入
- 结果页增加可信度提示，依据输入完整度与分析模式给出使用建议

### Changed

- 健康检查版本号升级为 `v1.5.0`
- 文案语气统一，前端加载态与结果页核心描述风格收口

### Notes

- 保持 `POST /api/fortune/analyze` 接口结构稳定，仅调整前端静态文案与健康检查版本
- 本次已本地 `docker compose up -d --build hxz-fortune` 构建验证通过

## v1.4.0 - 2026-03-31

### Added

- 历史记录摘要信息增强，展示模式与分析区间
- 基于历史记录的轻量结果对比
- 输入页快捷时间段入口
- 历史记录支持删除单条记录与清空全部记录
- 结果页新增“本次差异总结”，帮助解释当前结果与对比记录为什么不同
- 结果页新增历史记录概览，展示最近记录数量与模式分布
- `docs/releases/v1.4.md`

### Changed

- 健康检查版本号升级为 `v1.4.0`
- 项目当前阶段从 `v1.3` 切换到 `v1.4`
- `v1.4` 从第一批基础能力推进到第二批复盘与管理增强

### Notes

- 本次继续保持分析接口结构稳定，重点增强前端复盘与复用能力
- 已完成本地 `docker compose up -d --build hxz-fortune` 构建与运行验证
